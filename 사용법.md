# 바둑 기보 저장기 — 사용 설명서

바둑 기보를 기록하고, 리플레이하고, 다양한 변화를 연구할 수 있는 웹 앱입니다.  
PC, 태블릿, 스마트폰 브라우저에서 모두 사용할 수 있습니다.

---

## 화면 구성

```
┌─────────────────────────┬──────────────┐
│                         │  착수 정보   │
│                         │  리플레이    │
│       바둑판 (80%)      │  접바둑 설정 │
│                         │  컨트롤      │
│                         │  착수 기록   │
│                         │  내보내기    │
│                         │  불러오기    │
└─────────────────────────┴──────────────┘
```

좌측: 바둑판 / 우측: 기능 패널 (스크롤 가능)

---

## 1. 기보 기록하기

### 기본 착수
바둑판의 교점을 탭(클릭)하면 흑→백→흑 순서로 자동 교대합니다.

- 우측 상단 숫자: 현재까지 착수한 총 수
- 돌 위에 표시된 숫자: 해당 돌의 착수 순서
- 이미 돌이 놓인 자리는 착수 불가

### 사석 처리
상대 돌을 포위해 활로가 0이 되면 자동으로 사석 처리됩니다.

- 사석이 된 자리에는 해당 착수 번호만 점선 원으로 남음
- 같은 자리에 사석이 여러 번 발생하면 가장 최근 번호만 표시
- 새로운 돌이 그 자리에 놓이면 새 돌 번호로 덮어씌워짐

### 사석 카운터
착수 정보 패널 하단에 실시간으로 표시됩니다.

- ⚫ 흑 사석: 흑이 따낸 백 돌 수
- ⚪ 백 사석: 백이 따낸 흑 돌 수

---

## 2. 접바둑 두기

단계별로 진행합니다.

1. 우측 패널 **접바둑 설정**에서 돌 수 입력 (2~9점)
2. **접바둑 시작** 버튼 클릭
3. 흑 돌이 표준 성점 위치에 자동 배치됨
4. 이후 백 선착으로 게임 시작

접바둑 돌은 금색 테두리로 구별되며, 무르기 대상에서 제외됩니다.

---

## 3. 무르기 / 초기화

| 버튼 | 동작 |
|---|---|
| ↩ 1수 무르기 | 마지막 착수 1개 취소. 해당 수로 사석된 돌은 복원됨 |
| × 초기화 | 기보 전체 삭제, 처음부터 시작 |

---

## 4. 리플레이

기록된 기보를 한 수씩 앞뒤로 이동하며 복기할 수 있습니다.

### 버튼 설명

| 버튼 | 동작 |
|---|---|
| ⏮ | 첫 번째 수로 이동 |
| ◀ | 한 수 뒤로 |
| ▶ | 한 수 앞으로 |
| ⏭ | 마지막 수로 이동 |

### N수로 이동
특정 수로 바로 이동하고 싶을 때 사용합니다.

1. 입력창에 이동할 수 번호 입력 (예: 15)
2. **이동** 버튼 클릭

리플레이 중에는 우측 상단에 **현재 수 / 전체 수** 가 표시됩니다.  
예: `15 / 48` → 48수 기보의 15번째 수를 보는 중

---

## 5. 분기 기보 (변화 연구)

불러온 기보의 특정 시점부터 다른 수를 두어 변화를 연구할 수 있습니다.

1. 기보를 불러온 후 리플레이로 원하는 수로 이동
2. 그 시점에서 바둑판에 새로운 돌을 착수
3. **✦ 분기 기보 모드** 배지가 표시됨
4. 원하는 변화를 끝까지 두고 JSON으로 저장

분기 기보는 원본과 별개의 새로운 기보로 저장됩니다.

---

## 6. 기보 저장하기

### JSON 다운로드
기보 전체를 JSON 파일로 저장합니다. 나중에 불러와 리플레이할 수 있습니다.

파일명 형식: `baduk-kibo-YYYYMMDD-HHmm.json`

저장 형식 예시:
```json
[
  { "순서": 1, "색": "흑", "좌표": "D-4" },
  { "순서": 2, "색": "백", "좌표": "Q-16" },
  { "순서": 3, "색": "흑", "좌표": "D-16", "사석": true, "사석처리수": 10 }
]
```

### 클립보드 복사
JSON 데이터를 클립보드에 복사합니다. 메모앱이나 문서에 붙여넣기 가능합니다.

### 이미지 저장 (JPG)
현재 바둑판 상태를 이미지로 저장합니다.

- A~T, 1~19 좌표 레이블 포함
- 착수 번호, 사석 ghost 번호 모두 포함
- 우측 하단에 날짜/시간/총 수 워터마크 자동 삽입

---

## 7. 기보 불러오기

저장해 둔 JSON 파일을 불러와 기보를 복원합니다.

1. **📂 JSON 파일 선택...** 버튼 클릭
2. 저장된 `.json` 파일 선택
3. 기보가 자동으로 복원됨

불러온 후에는 리플레이 모드로 진입합니다.  
접바둑 돌이 포함된 기보는 자동으로 접바둑 배지가 표시됩니다.

---

## 8. 착수 기록

우측 패널 착수 기록 목록에서 전체 기보를 확인할 수 있습니다.

| 표시 | 의미 |
|---|---|
| ● | 흑 |
| ○ | 백 |
| 황금색 행 | 접바둑 돌 |
| 파란색 행 + 사석(N수) | 사석 처리된 돌 (N번째 수에서 제거됨) |

---

## 9. 좌표 체계

이 앱은 표준 바둑 좌표 표기를 사용합니다.

- 열: A~T (I 제외, 총 19열)
- 행: 1~19 (아래에서 위로)
- 표기 예: `D-4`, `Q-16`, `K-10`

## 자주 묻는 질문

**Q. 자충수를 두면 어떻게 되나요?**  
활로가 없는 위치에는 착수 자체가 되지 않습니다. (소리 없이 무시됨)

**Q. 무르기를 여러 번 하고 싶으면?**  
↩ 버튼을 여러 번 누르거나, 리플레이의 ◀ 버튼으로 원하는 수까지 이동하세요.

**Q. 리플레이 중에 새 돌을 두면?**  
해당 시점부터 분기 기보가 시작됩니다. ✦ 배지가 표시되고 새 기보로 저장할 수 있습니다.

**Q. 접바둑 돌은 무르기가 되나요?**  
접바둑 돌은 게임 시작 전에 배치되는 돌이므로 무르기 대상에서 제외됩니다.

**Q. JSON 파일은 어디에 저장되나요?**  
기기의 기본 다운로드 폴더에 저장됩니다. iPad/iPhone의 경우 파일 앱에서 확인할 수 있습니다.
