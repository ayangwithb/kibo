## JSON 파일 형식

기보를 저장하면 다음 형식의 JSON 파일이 생성됩니다.

### 전체 구조

```json
{
  "흑": "Anna",
  "백": "이명진",
  "날짜": "2026-05-25",
  "착수": [ ... ]
}
```

| 필드 | 타입 | 설명 |
|---|---|---|
| `흑` | string | 흑 플레이어 이름. 미입력 시 `"none"` |
| `백` | string | 백 플레이어 이름. 미입력 시 `"none"` |
| `날짜` | string | 대국 날짜 (YYYY-MM-DD). 미입력 시 저장 당일 날짜 자동 기입 |
| `착수` | array | 착수 기록 배열 (순서대로) |

### 착수 항목 구조

```json
{ "순서": 1, "색": "흑", "좌표": "D-4" }
```

| 필드 | 타입 | 설명 |
|---|---|---|
| `순서` | number | 착수 순서 (1부터 시작) |
| `색` | string | `"흑"` 또는 `"백"` |
| `좌표` | string | 착수 위치. 열(A~T, I 제외) + 행(1~19). 예: `"D-4"`, `"Q-16"` |
| `유형` | string | 접바둑 돌이면 `"접바둑"` (해당 없으면 생략) |
| `사석` | boolean | 사석 처리된 돌이면 `true` (해당 없으면 생략) |
| `사석처리수` | number | 몇 번째 착수에서 사석 처리됐는지 (사석인 경우만 존재) |

### 전체 예시

```json
{
  "흑": "Anna",
  "백": "none",
  "날짜": "2026-05-25",
  "착수": [
    { "순서": 1, "색": "흑", "좌표": "D-4" },
    { "순서": 2, "색": "백", "좌표": "Q-16" },
    { "순서": 3, "색": "흑", "좌표": "D-16" },
    { "순서": 4, "색": "백", "좌표": "Q-4" },
    { "순서": 5, "색": "흑", "좌표": "K-10" },
    { "순서": 6, "색": "백", "좌표": "C-3", "사석": true, "사석처리수": 12 },
    { "순서": 7, "색": "흑", "좌표": "R-4", "유형": "접바둑" }
  ]
}
```

### 접바둑 기보 예시

접바둑 돌은 `"유형": "접바둑"` 필드로 구별됩니다. 착수 순서 초반에 흑으로 연속 기록되며, 이후 백 선착으로 게임이 이어집니다.

```json
{
  "흑": "입문자",
  "백": "선생님",
  "날짜": "2026-05-25",
  "착수": [
    { "순서": 1, "색": "흑", "좌표": "D-4",  "유형": "접바둑" },
    { "순서": 2, "색": "흑", "좌표": "Q-16", "유형": "접바둑" },
    { "순서": 3, "색": "흑", "좌표": "D-16", "유형": "접바둑" },
    { "순서": 4, "색": "흑", "좌표": "Q-4",  "유형": "접바둑" },
    { "순서": 5, "색": "백", "좌표": "K-10" }
  ]
}
```

### import 시 호환 형식

구버전 plain array 형식도 불러오기 가능합니다. 단, 대국 정보(흑/백/날짜)는 복원되지 않습니다.

```json
[
  { "순서": 1, "색": "흑", "좌표": "D-4" },
  { "순서": 2, "색": "백", "좌표": "Q-16" }
]
```

### 좌표 체계 참고

열(A~T)은 I를 제외한 19개 알파벳을 사용합니다.

```
A B C D E F G H J K L M N O P Q R S T
1 2 3 4 5 6 7 8 9 ...               19
```

행은 바둑판 아래(1선)에서 위(19선) 방향으로 번호가 매겨집니다.
